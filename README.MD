Hellgile Telegram Game

1. Пользователь открыл бот впервые
    1. Для пользователя сгенерирована и сохранена в БД новая игра (проект)
        1. Сгенерировано название проекта
        1. Сгенерированы основные показатели проекта (стоимость, длительность)
        1. Сгенерирован список сотрудников
        1. Данные сохранены в БД для id клиента
    1. Пользователю отправлено приветственное сообщение и главный экран
    1. Пользователь начал игру
1. Пользователь планирует спринт
    1. Пользователю отправляется главный экран
1. Спринт начался
    1. Обсчет спринта
        1. Для каждой задачи...
1. Спринт завершился
1. Переход к пункту "Пользователь планирует спринт"

Структура проекта:
- Проект (Project)
    - Глобальные константы
        - Наименование проекта (name)
            * Выбирается в начале проекта случайно из предзаполненного списка
        - Используемая валюта (currency)
            * Выбирается в начале проекта случайно из предзаполненного списка
        - Длина спринта в днях (workingDaysInSprint)
            * Фиксированное значение: [10]
    - Целочисленные переменные
        - Деньги
            - Исходная оценка проекта (valuation)
                * Диапазон: [1000000;30000000]
                * Исходное значение: [1000000;30000000]
            - Остаток (money)
                * Диапазон: [0;30000000]
                * Исходное значение: [1000000;30000000]
        - Время
            - Исходная длительность проекта в спринтах (totalSprints)
                * Диапазон: [10;300]
                * Исходное значение: [10;300]
            - Текущий спринт (currentSprint)
                * Диапазон: [1;totalSprints]
                * Исходное значение: [1]
            - Остаток спринтов (sprintsLeft)
                * Диапазон: [1;totalSprints]
                * Исходное значение: [totalSprints - 1]
    - Сущности
        - Заказчик
        - Список задач
            - Изначально запланированные задачи
                * Генерируются в начале проекта случайно в зависимости от valuation проекта
            - Невыявленные задачи (ошибки в коде, падение железа и т.п.)
                * Изначально отстутствуют
                * Добавляются в процессе работы над задачами сотрудниками
                * Добавляются заказчиком в зависимости от [attentionFactor * relationshipFactor]
            - Задачи в работе
                * Добавляются в список по мере назначения задач сотрудникам
            - Выполненные задачи
                * Добавляются в список после выполнения
        - Список сотрудников

- Сущности
    - Заказчик
        - Имя заказчика (name)
            * Выбирается в начале проекта случайно из предзаполненного списка            
        - Уровень внимания к проекту (attentionFactor) - вероятность появления новых задач от заказчика по мере работы
            * Диапазон: [0.00;1.00]
            * Исходное значение: [0.00;1.00]
        - Отношения с заказчиком (relationshipFactor) - вероятность отмены ранее созданной задачи заказчиком
            * Диапазон: [0.00;1.00]
            * Исходное значение: [0.50;1.00]    
    - Сотрудник
        - Имя (name)
            * Выбирается при генерации случайно из предзаполненного списка
        - Специализация (class)
            * Выбирается при генерации случайно из предзаполненного списка
            * Диапазон: [DEVELOPER,TESTER,ACCOUNT_MANAGER]
        - Уровень (level)
            * Диапазон:
                * JUNIOR: [1;100]
                * MIDDLE: [101:200]
                * SENIOR: [201:300]
                * LEAD: [301;400]
            * Исходное значение: [1;400]
        - Стоимость (salary)
            * Рассчитывается при генерации случайно в зависимости от уровня и специализации сотрудника
            * Диапазон базовой стоимости:
                * JUNIOR: [20000;50000]
                * MIDDLE: [50000;80000]
                * SENIOR: [80000;120000]
                * LEAD: [120000;200000]
            * Коэффициент по специализации:
                * DEVELOPER: [1.0]
                * TESTER: [0.8]
                * ACCOUNT_MANAGER: [2.0]
        - Темперамент (temperament)
            * Выбирается при генерации случайно из предзаполненного списка
            * Диапазон: [SANGUINE,PHLEGMATIC,CHOLERIC,MELANCHOLIC]
        - Ответственность (responsibility)
            * Диапазон: [0.00;1.00]
            * Исходное значение: [0.00;1.00]
        - Настроение (mood)
            * Диапазон: [0.00;1.00]
            * Исходное значение: [1.00]
        - Лояльность сотрудника (loyalty)
            * Диапазон: [0.00;1.00]
            * Исходное значение: [1.00]          
    - Задача
        - Название задачи (name)
            * Выбирается в начале проекта случайно из предзаполненного списка
        - Тип задачи (type)
            * Устанавливается при создании задачи
            * Диапазон: [PLANNED,NEW_FEATURE,ISSUE]
        - Сложность (level)
            * Рассчитывается при генерации случайно
            * Диапазон: [JUNIOR,MIDDLE,SENIOR,LEAD]
        - Требуемые навыки (skills)
            * Выбирается в начале проекта случайно из предзаполненного списка (включая пустое значение)
        - Текущая специализация (class)
            * Меняется в процессе выполнения задачи
            * Диапазон: [DEVELOPER,TESTER,ACCOUNT_MANAGER]
            * Исходное значение: [DEVELOPER]
            * Модель изменения в зависимости от типа: 
                * PLANNED: [DEVELOPER -> TESTER]
                * NEW_FEATURE: [ACCOUNT_MANAGER -> DEVELOPER -> TESTER]
                * BUG: [TESTER -> DEVELOPER -> TESTER] или [ACCOUNT_MANAGER -> TESTER -> DEVELOPER -> TESTER]
        - Текущий статус (status)
            * Меняется в процессе выполнения задачи
            * Диапазон: [NEW,ASSIGNED,DONE]
            * Модель изменения: [NEW -> (ASSIGNED -> DONE)]                
        - Текущий сотрудник (employee)
            * Устанавливается в спринте
        - Количество запланированных функциональных элементов (plannedStoryPoints)
        - Количество выполненных функциональных элементов (completedStoryPoints)
   
Задача 

Задача сводится ко времени (оценка)

Длительность (количество функциональных элементов) - SP, где базовый 1 SP = 1 H
    Генерируем случайное число - 1.0;10.0
Сложность (джун..лид)
    Джун-джун - 1.0
    Джун-мидл - 2.0
    Джун-син - 3.0
    Джун-лид - 4.0
    
    Мидл-джун - 1.0
    Мидл-мидл - 1.0
    Мидл-син - 2.0
    Мидл-лид - 3.0
    
    Син-джун - 1.0
    Син-мидл - 1.0
    Син-син - 1.0
    Син-лид - 2.0
    
    Лид-джун - 1.0
    Лид-мидл - 1.0
    Лид-син - 1.0
    Лид-лид - 1.0
Масштабируемость (джун и лид будут делать одинаково долго)
    
Зависимость от скилов (коэффициент эффективности конкретного сотрудника?)
    Нет скила - 2.0
    Есть скил - 1.0
Time = SP * level * skill
